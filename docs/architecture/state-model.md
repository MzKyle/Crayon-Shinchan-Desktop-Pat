# 状态模型

## 物理状态

`PetPhysics.gd` 的 `state` 是桌宠位置更新的核心状态：

| 状态 | 进入条件 | 行为 | 退出条件 |
| --- | --- | --- | --- |
| `Idle` | 启动、落地结束、手动停止 | 不主动移动 | 行为调度、拖拽、菜单 |
| `Grabbed` | 长按触发抱起 | 窗口位置弹簧跟随鼠标 | 释放 |
| `Flinging` | 快速释放 | 带速度飞出，受重力和阻尼影响 | 落地、撞墙、速度衰减 |
| `Falling` | 慢速释放且重力开启 | 下落 | 落地 |
| `Landing` | 低速触地 | 触发落地压缩 | 回到 `Idle` |
| `Walk` | 手动或活泼模式散步 | 横向移动，碰边反弹 | 手动动作或行为调度 |
| `WallAttached` | 慢速撞墙吸附 | 贴在屏幕左右边缘 | 开始贴边走或其他动作 |
| `EdgeWalk` | 活泼模式贴边行为 | 沿屏幕边上下移动 | 用户交互或行为调度 |
| `Peeking` | 拖到屏幕边缘释放 | 窗口缩小，只显示偷看图 | 点击或拖拽唤回 |

## 行为模式

| 模式 | 自动行为 | 适合场景 |
| --- | --- | --- |
| 安静 | 不自动散步、不贴边、不捣乱 | 工作、开会、录屏 |
| 活泼 | 随机散步、贴边走、邀请玩、脚印特效 | 日常桌面陪伴 |
| 捣乱 | 每 20-40 秒触发一次温和抢鼠标视觉演出 | 娱乐和展示 |

切到捣乱模式时会先快速试探一次，之后按常规定时触发。捣乱演出不会移动、锁定或修改系统鼠标位置，只让桌宠窗口贴近鼠标并播放“费力抢鼠标”的视觉效果。

## 输入状态

`InteractionController.gd` 通过两个计时器区分点击、双击和长按：

- `click_timer`：短暂延迟单击触发，避免和双击冲突
- `long_timer`：按下超过 `0.35s` 后进入抱起

释放时会从最近若干鼠标采样计算速度。速度超过阈值时进入甩飞，否则按轻放处理。

## 截图贴图状态

截图贴图模块维护：

| 字段 | 说明 |
| --- | --- |
| `config` | 快捷键、截图后端、最大贴图数 |
| `history` | 最近截图文件，最多 3 张 |
| `pins` | 当前打开的贴图窗口 |
| `active_pin` | 最近点击或拖动过的贴图 |
| `hotkey_pid` | X11 全局快捷键辅助进程 PID |
| `screenshot_active` | 防止并发截图 |

贴图数量超过配置上限时会拒绝继续贴图，避免桌面被贴图窗口铺满。
