# 行为系统

## 行为调度器

`BehaviorBrain.gd` 是一个轻量定时器驱动的行为调度器。它不直接移动角色，只发出请求：

- `action_requested("walk")`
- `action_requested("idle")`
- `action_requested("edge")`
- `action_requested("invite")`
- `mischief_requested("grab")`
- `mischief_requested("footprint")`

`Main.gd` 接到请求后，会判断当前是否忙碌，再决定是否执行。

## 安静模式

安静模式仍然运行计时器，但每次触发后只重新排程，不发出行为请求。

适合：

- 工作时保持桌面干净
- 录屏或演示
- 只想手动互动

## 活泼模式

活泼模式每 4-8 秒随机决定一次行为：

| 随机区间 | 行为 |
| --- | --- |
| `< 0.34` | 散步 |
| `< 0.52` | 回到闲置 |
| `< 0.66` | 贴边走 |
| `< 0.80` | 邀请玩 |
| 其余 | 脚印小特效 |

这些行为只在不忙碌时执行。抱起、小游戏、偷看和捣乱演出都会让普通行为暂停或跳过。

## 捣乱模式

捣乱模式的核心是“费力抢鼠标”视觉演出：

- 切换到捣乱模式后约 1 秒试探一次
- 后续每 20-40 秒触发
- 演出持续 4 秒
- 角色窗口贴近鼠标并抖动
- 画出拉扯线和汗滴
- 右上角显示“停”按钮

它不会移动系统鼠标，也不会锁定鼠标。鼠标穿透多边形会被收缩到“停”按钮区域，避免演出期间影响用户操作桌面。

## 忙碌判断

`Main.gd` 的忙碌状态包括：

- 正在抱起或飞行
- 正在小游戏
- 正在偷看
- 正在捣乱演出

行为调度请求必须通过忙碌判断，避免多个状态互相覆盖。
